`godot-game-one` 里没有任何 `.wav`、`.ogg`、`.mp3` 文件。所有音效——射击、爆炸、受击、UI 提示——全部在运行时用 GDScript 计算生成。

## 三种波形，覆盖全部场景

整个音效系统只用了三种基础波形：

| 波形 | 原理 | 用途 |
|------|------|------|
| **Tone（正弦波）** | `sin(phase * TAU)` + 指数衰减 | UI 音效、受击反馈 |
| **Sweep（频率扫描）** | 频率线性插值 `lerp(start, end, t)` | 射击、敌人死亡 |
| **Noise（白噪声）** | `randf_range(-1, 1)` + 快速衰减 | 爆炸 |

每种波形都写入 `AudioStreamWAV` 的 16-bit PCM buffer，约 0.1~0.3 秒长度。7 个游戏音效就是这三种波形的不同参数组合：

```
射击：Sweep(2000→500, 0.1s)    — 频率下行，像激光
爆炸：Noise(0.4s)              — 纯噪声，短促有力
敌人死亡：Sweep(800→200, 0.2s) — 低沉的下行扫频
受击：Tone(600, 0.15s)         — 单音提示
UI确认：Tone(880, 0.08s)       — 高频短促
```

## BGM：四音符循环

背景音乐同样是程序化生成。用一个简单的音符序列定义旋律：

```gdscript
var notes = [
    [261.63, 0.5],  # C4, 半拍
    [329.63, 0.5],  # E4
    [392.00, 0.5],  # G4
    [349.23, 0.5],  # F4
]
```

每个音符生成一段 Tone，首尾拼接成一个 `AudioStreamWAV`，设置 `loop_mode` 为循环。游戏 BGM 和 Boss BGM 各一套音符数组，战斗时热切换。

## 为什么不用音频文件？

1. **零依赖**：不需要找素材、处理版权、管理资源文件
2. **体积极小**：APK 不用打包任何音频资源
3. **参数可调**：想让射击音更高？改一个频率数值的事
4. **学习成本低**：不需要音频编辑器，不需要理解音频格式

## 局限

程序化音效的表现力有限——没有混响、没有和声、没有复杂的音色层次。对于复古风格的弹幕游戏来说，这种"干净的电子感"恰好是风格的一部分。但对于需要沉浸式音效的项目，还是得上专业音频方案。

> 最好的工程方案不是最强的，是最适合当前阶段的。
