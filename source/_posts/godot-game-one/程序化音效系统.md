## 三种波形，覆盖全部场景

整个音效系统只用了三种基础波形，全部通过计算 PCM 采样值实现：

| 波形 | 原理 | 用途 |
|------|------|------|
| **Tone（正弦波）** | `sin(phase * TAU)` + 指数衰减 | UI 音效、受击反馈 |
| **Sweep（频率扫描）** | 频率线性插值 `lerp(start, end, t)` | 射击、敌人死亡 |
| **Noise（白噪声）** | `randf_range(-1, 1)` + 快速衰减 | 爆炸 |

核心实现（Tone 生成器）：

```python
func _generate_tone(freq: float, duration: float, volume: float = 0.3) -> AudioStreamWAV:
    var num_samples := int(sample_rate * duration)
    var stream := AudioStreamWAV.new()
    stream.format = AudioStreamWAV.FORMAT_16_BITS
    stream.mix_rate = sample_rate
    var data := PackedByteArray()
    data.resize(num_samples * 2)

    var phase := 0.0
    var phase_step := freq / sample_rate
    var decay := 1.0
    var decay_step := 1.0 / num_samples

    for i in num_samples:
        decay = maxf(decay - decay_step * 1.2, 0.0)
        var sample_value := sin(phase * TAU) * volume * decay
        var int_sample := int(clampf(sample_value, -1.0, 1.0) * 32767)
        data[i * 2] = int_sample & 0xFF
        data[i * 2 + 1] = (int_sample >> 8) & 0xFF
        phase = fmod(phase + phase_step, 1.0)

    stream.data = data
    return stream
```

Sweep 和 Noise 的实现几乎相同，区别仅在于核心采样公式。7 个游戏音效就是这三种波形的不同参数组合：

```python
# SFX 单例中的公开接口
func play_shoot():        _play(_generate_sweep(2000, 500, 0.1))
func play_explosion():    _play(_generate_noise(0.4))
func play_enemy_death():  _play(_generate_sweep(800, 200, 0.2))
func play_player_hurt():  _play(_generate_tone(600, 0.15))
func play_ui_confirm():   _play(_generate_tone(880, 0.08))
func play_ui_select():    _play(_generate_tone(660, 0.05))
```

## BGM：音符序列 + 循环

背景音乐同样是程序化生成。用一个音符序列定义旋律，每个音符是一个 `[频率, 时值]` 对：

```python
var notes = [
    [261.63, 0.5],  # C4, 半拍
    [329.63, 0.5],  # E4
    [392.00, 0.5],  # G4
    [349.23, 0.5],  # F4
]
```

生成流程：每个音符调用 `_generate_tone()` 生成一段 PCM 数据，首尾拼接成一个完整的 `AudioStreamWAV`，设置 `loop_mode` 为循环。游戏 BGM 和 Boss BGM 各一套音符数组，战斗时通过 `BGM.play_boss_bgm()` / `BGM.play_bgm()` 热切换。

## 为什么不用音频文件？

1. **零依赖**：不需要找素材、处理版权、管理资源文件
2. **体积极小**：APK 不用打包任何音频资源
3. **参数可调**：想让射击音更高？改一个频率数值的事
4. **学习成本低**：不需要音频编辑器，不需要理解音频格式

## 局限与适用场景

程序化音效的表现力有限——没有混响、没有和声、没有复杂的音色层次。但对于复古风格的弹幕游戏来说，这种"干净的电子感"恰好是风格的一部分。

**适合用程序化音效的场景**：
- Game Jam / 原型验证阶段，不想花时间找素材
- 复古 / 像素风格游戏，音效本身就是"低保真"美学
- 嵌入式 / 极小体积要求的项目

**不适合的场景**：
- 需要沉浸式音效的 3A / 叙事向游戏
- 需要多层混音、空间音频的 VR/AR 项目
- 音效是核心卖点的音乐游戏

> 最好的工程方案不是最强的，是最适合当前阶段的。
